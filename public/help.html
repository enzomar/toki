📊 Two Forecasting Approaches

📐 Deterministic

The "plan" number.

  • Multiplies each agent's tokens × calls × traffic share
  • Gives one fixed value — the expected outcome
  • Assumes everything goes exactly as configured
  • Fast, predictable, easy to explain

"If nothing surprising happens, it'll cost this much."

🎲 Monte Carlo

The "reality" range.

  • Simulates hundreds of real conversations
  • Each run takes random paths through your agents
  • Varies RAG chunks, MCP calls, retries each time
  • Produces a range: typical, high, and worst-case

"Here's what it could actually cost — including bad days."

Each simulation run:

  1. Start at entry agents
  2. Roll dice at each connection (probability = edge weight)
  3. If "yes" → follow that path, if "no" → skip it
  4. At each agent, randomly vary token counts (±15%)
  5. RAG: re-sample how many chunks, how large
  6. MCP: chance of tool chaining and retries
  7. Calculate cost using actual model pricing

What you learn:

  • p50 — Typical cost (half are lower)
  • p90 — High-end (only 10% exceed this)
  • p99 — Tail risk (1 in 100 conversations)
  • Tail risk factor — How much worse p99 is vs p50
  • Dominant nodes — Which agents drive cost spikes

A tail risk of 2.5× means your worst-case day costs 2.5× your typical day.

Alignment check: When deterministic ≈ MC expected (within ±15%), your model is reliable.
When they diverge ⚠, real-world variability might significantly exceed your plan.

🎯 Confidence Score

The confidence % tells you how much to trust the forecast.

What reduces confidence:

Misalignment (MC ≠ deterministic)−20%
High variance (results spread widely)−10% to −40%
Default token values (not measured)−15%
Missing connections (routing unknown)−15%

How to improve it:

  • Measure real prompt token counts (use Token Tool)
  • Set history growth for multi-turn agents
  • Add connections between agents with weights
  • Increase simulation count (200→500)
  • Configure MCP call counts accurately

⚠️ Scope & Limitations

✅ What it does
Planning-stage cost forecasting for multi-agent LLM systems using deterministic + Monte Carlo simulation
❌ What it doesn't do
Production billing reconciliation, real-time spend monitoring, or actual API call execution
📈 Roadmap
Usage log calibration, scenario comparisons, team workspaces, budget alerts

✅ Implemented

  • Monte Carlo simulation — graph-walking with random paths
  • Connection weights as probabilities — edges are Bernoulli gates
  • Per-node cost tracking — actual model pricing per agent
  • MCP chain/retry depth — geometric probability model
  • Cache pricing — model-specific cached token rates
  • Batch pricing — optional 50% discount toggle
  • Volume discounts — global % discount for enterprise
  • Configurable variance — all simulation factors tunable
  • Routing modes — weighted, fan-out, round-robin
  • History growth — geometric series per-call growth

⚠️ Not yet modeled

  • Rate limit / 429 retries — throttling cost not simulated
  • Fine-tuning cost — training epochs excluded
  • Dynamic agent spawning — topology is static at compile time
  • Latency estimation — only token/cost, not time-to-response
  • Cross-region pricing — no per-region rate differences
  • Token counting accuracy — uses estimates, not real tokenizer
